Why this exists

The internet
forgets.

Independent music lives and dies by the platforms that host it.

Platforms come and go

Bandcamp has changed hands. Before that, MySpace took entire catalogs with it. SoundCloud nearly shut down in 2017. Every platform that hosts music is a single point of failure for the artists who trusted it.

Independent artists are most at risk

Major label releases have redundant distribution and institutional backing. Independent artists often have no copy anywhere else. When the page goes, the music goes.

A web archive captures everything

Not just an MP3. The artwork at full resolution, the release notes, the credits, the tags, the story behind the record, frozen exactly as it was, replayable years from now without any dependency on Bandcamp.

This is small, personal, and imperfect

This project archives one person's followed artists. It is not comprehensive. But comprehensive efforts start somewhere, and an archived record is better than none.

How it works

From Bandcamp
to permanent storage.

01

Metadata is fetched and tracked

For each followed artist, the pipeline scrapes release metadata (titles, tags, credits, artwork URLs, Bandcamp IDs) and stores it locally as JSON. This becomes the source of truth for everything downstream.

fetch_metadata.py
02

Browsertrix crawls each release

A custom browser behavior runs inside a Browsertrix container and reads audio stream URLs directly from the page's embedded JSON. Each MP3 is fetched as a complete file, bypassing Bandcamp's chunked streaming, so the full audio is captured in the archive.

crawl.py · behaviors/bandcamp.js
03

A WACZ file is produced

The crawl output is a WACZ, a self-contained web archive containing the full page, all artwork, and every audio stream. It can be replayed offline using ReplayWeb.page without any dependency on Bandcamp still existing.

WACZ · ReplayWeb.page
04

Uploaded to Pixeldrain

Each WACZ and its JSON sidecar are uploaded to Pixeldrain as a permanent file. The Pixeldrain file ID is written back to the local metadata so the browser can link directly to it.

upload.py · pixeldrain.com
05

New releases trigger automatically

An optional IMAP daemon watches for Bandcamp new-release notification emails and triggers the pipeline immediately when one arrives, no manual intervention needed for followed artists.

email_watcher.py · IMAP IDLE

The project

Two parts
working together.

Archiver

Bandcamp WACZ Archiver

The pipeline. Fetches metadata, orchestrates Browsertrix crawls, produces WACZ files, and uploads them to Pixeldrain. Supports single albums, full discographies, and automatic email-triggered archiving.

github.com/Bandcamp-Web-Archive/bandcamp-wacz-archiver

Browser

Bandcamp Archive Browser

A static Jekyll site that turns the archiver's JSON output into a searchable, filterable catalog, with live archive status badges, tag filtering, cover art, and direct links to every file on Pixeldrain. Updated at least once a day.

bandcamp-web-archive.github.io/browse
